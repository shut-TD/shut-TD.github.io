# 探索回调函数

  • 有关回调函数做后门的利用已经是十分常见的了，以及关于部分回调函数后门如何使用也已经被重复讲过了，因此这篇文章的着重点主要还是放在给我一个函数如何去判断可以作为回调函数后门，以及如何能够自己去找到这样的函数并利用。

  • 首先还是先讲讲个人对于回调函数的理解，无非就是参数位有一个能够是自定义函数名称或者 php 内置函数名称，有一个或多个其他参数位的参数作为回调函数的参数，所以回调函数有做后门的潜力就是能够控制一个函数以及传递给它的参数，并且该函数仅用名称就可以。

# 如何判断该函数是回调函数后门？

  • 先说结论：
    1、有 callback
    2、回调参数支持字符串类型
    3、该函数返回值支持字符串 / 数组类型 (有回显)
    如果满足以上三个特征，那么这个函数基本上就能作为回调函数后门，如何判断这三个特征？直接访问 https://www.php.net/ 搜索该函数，在这里我们举一个十分典型的案例：call_user_func


其回调参数也发现没有做什么限制，并且该函数返回值是 mixed 也就是啥类型都可以，因此该函数就是回调函数后门。

我们再来提一个不那么常见的函数，来进行观察：array_reduce

现在我们对比一下虽然满足有 callback，但不是回调函数后门，这里举一个例子：usort

可以很明显的发现，该函数的返回值固定为 true 那肯定无法直接将命令回显出来，所以这并不是回调函数后门。

# 总结

相信看完这几个函数的详细对比，你已经掌握了基本的如何去寻找或者判断一个回调函数后门了，这里需要注意的是这里并没有考虑非回显以及嵌套回调的情况因此可能会出现误解，还是自行仔细研究，我仅仅只做牵引的效果，这里贴几个回调函数进行测试 (以下都是可直接回显的回调后门函数)。
